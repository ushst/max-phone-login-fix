# Текст для карточки в Chrome Web Store

Готовые поля — просто скопировать в форму на https://chrome.google.com/webstore/devconsole

## Название (Title, до 45 символов)
Вход по номеру для MAX (неофициально)

## Краткое описание (Summary, до 132 символов)
Показывает скрытую кнопку «Войти по номеру» на web.max.ru, если её перекрывает QR-код. Данные не собирает.

## Категория
Productivity (Продуктивность) — или Accessibility, если такая категория доступна в вашем регионе.

## Язык
Русский (основной), можно добавить английскую локализацию позже через `_locales`.

## Подробное описание (Description)

```
Неофициальное расширение для web.max.ru.

Проблема: на странице входа в веб-версию мессенджера MAX по умолчанию
показывается только QR-код, а ссылка «Войти по номеру телефона» — мелкая
подпись под ним, которую легко не заметить или которая визуально
перекрывается блоком с QR-кодом.

Что делает расширение:
• находит ссылку «Sign in with phone number» на странице;
• снимает с неё ограничения по видимости/клику, если её перекрывает
  другой элемент;
• добавляет постоянно видимую кнопку «Войти по номеру телефона» внизу
  экрана — один клик, и открывается обычная форма ввода номера с SMS-кодом.

Расширение не собирает и не передаёт никакие данные, не запрашивает
никаких разрешений (permissions) в браузере, работает только на домене
web.max.ru и не имеет доступа к другим сайтам, вкладкам или истории.

Это независимый инструмент, не связанный с разработчиками MAX и не
являющийся официальным продуктом.
```

## Обоснование единственного назначения (Single purpose)
Расширение решает ровно одну задачу: делает доступной уже существующую
на сайте web.max.ru функцию входа по номеру телефона, когда она визуально
скрыта или перекрыта QR-кодом.

## Обоснование permissions
В manifest.json не запрошено ни одного `permission` — используется только
`content_scripts` с `matches: ["https://web.max.ru/*"]`. В форме
"Permission justification" в Developer Dashboard можно указать:

```
Расширение не запрашивает permissions. Используется только
content_scripts, ограниченный одним доменом (web.max.ru), для показа
и активации уже существующего элемента интерфейса сайта.
```

## Privacy practices (вкладка в Developer Dashboard)
- Does this extension collect user data? — Нет (No).
- Все чекбоксы про personally identifiable information, health info,
  financial info, authentication info, personal communications,
  location, web history, user activity, website content — оставить
  не отмеченными (расширение ничего из этого не читает и не передаёт).
- Privacy policy URL: репозиторий уже на GitHub, можно использовать
  готовую ссылку —
  https://github.com/ushst/max-phone-login-fix/blob/main/PRIVACY.md
  (или raw-версию:
  https://raw.githubusercontent.com/ushst/max-phone-login-fix/main/PRIVACY.md)
